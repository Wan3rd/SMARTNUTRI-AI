Real-Time WebSockets Alerts, Error Notification Audits & Timezone Mismatch Fixes
  • ServerSet up **Socket.io** using private rooms mapped to parent user IDs. We exposed this socket manager as Express middleware to all routers.
  • ClientCoded the layout socket hook: parent dashboards instantly receive notification banners when a clinician reviews a meal, automatically refreshing stats in the background without manual page reloads.
  • ClientPolished `ReviewLogModal.jsx`: we close the modal *first* and wait 300ms before triggering the parent page update. This gives exit animations time to slide out smoothly and prevents the background page from shifting or jumping.

  • ClientAudited and upgraded all client-side form submissions and API catch blocks across 7 core pages and components (Settings.jsx, Profile.jsx, ClientDetails.jsx, ReviewLogModal.jsx, MealLogger.jsx, MealDetailModal.jsx, and RecipeDetail.jsx). All hardcoded fallback generic strings were replaced with dynamic extractions of the precise server-side validation error messages (using err.response?.data?.message) so users instantly know exactly what input field triggered a failure.
  • ClientFixed a subtle timezone mismatch shift bug in the MealLogger.jsx client submissions. Pre-parsed the timezone-naive local datetime string from the browser as local time and converted it to a standard UTC ISO string before POSTing, preventing Node.js from incorrectly parsing it as UTC and shifting the meal logs (and therefore calorie progress and the daily wave fluid tumbler) 8 hours into the future.
  • ServerUpgraded the /admin/stats router to accept a dynamic range parameter (week, month, or year) and group registration and log counts daily or monthly dynamically while padding gaps. We also added a server-side compiledAt synchronization timestamp.

They format timestamps dynamically in local format within glassmorphic tooltips, adjust the X-axis to the target range granularity, and display a pulsing live-update synchronized clock.
  • ServerCoded the GET /admin/diagnostics self-test API that runs concurrent, live pings to Prisma Database (measures raw execution query latency), Google Gemini AI API (handshakes using Process Key credentials), and Cloudinary Storage (validates token parameters).
  • ClientDesigned and mounted a dedicated PlatformDiagnostics.jsx diagnostic panel inside the System Telemetry monitoring console. It features one-click testing, pulsing progress states, and beautifully color-coded (Healthy/Failed) HSL cards showing live response speed stats.

Data Reliability Details

Safe Deletions: Removing logs, profiles, or linked caregivers is done inside database transactions. This ensures all child tables are cleared together, preventing broken database links.

Calorie Recalculations: The system guards your custom nutrition inputs. Ingredient calorie numbers are only recalculated when caregiver plate waste edits are made, preserving manual nutritionist overrides.
